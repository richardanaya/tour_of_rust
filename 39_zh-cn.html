丑陋的 Option/Result 处理

当你只是试图快速地写一些代码时，Option/Result 对付起来可能比较无聊。 Option 和 Result 都有一个名为 unwrap 的函数：这个函数可以简单粗暴地获取其中的值。 unwrap 会：

  1. 获取 Option/Result 内部的值
  2. 如果枚举的类型是 None/Err， 则会 panic!

这两段代码是等价的：

my_option.unwrap()

match my_option {
    Some(v) => v,
    None => panic!("some error message generated by Rust!"),
}

类似的：

my_result.unwrap()

match my_result {
    Ok(v) => v,
    Err(e) => panic!("some error message generated by Rust!"),
}

不过啊，做个好 Rustacean，正确地使用 match！
